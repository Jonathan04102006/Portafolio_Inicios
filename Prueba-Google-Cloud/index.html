Guía de Integración: Google Auth desde Cero

1. Configuración de la Infraestructura (Google Cloud)

Para que la aplicación funcione, se registró un proyecto en la consola de Google para obtener un Client ID único. Es vital que las URLs de redirección coincidan exactamente con el entorno local:

  • URL de Redirección Autorizada: http://127.0.0.1:5500/Prueba-Google-Cloud/Login/entrada.html
  • Origen de JavaScript: http://127.0.0.1:5500

2. Fase de Autenticación (Petición)

Utilizamos la API de Autorización de Google para solicitar el consentimiento del usuario.

ENDPOINT: https://accounts.google.com/o/oauth2/v2/auth

Código: main.js

const CLIENT_ID = '59625450780-pqobp1879riooqvlu2vpckh397q35fj6.apps.googleusercontent.com';
const REDIRECT_URI = 'http://127.0.0.1:5500/Prueba-Google-Cloud/Login/entrada.html';

const login = () => {
    const authUrl = new URL('https://accounts.google.com/o/oauth2/v2/auth');
    const params = {
        client_id: CLIENT_ID,
        redirect_uri: REDIRECT_URI,
        response_type: 'token',
        scope: 'openid email profile',
        prompt: 'select_account'
    };

    Object.keys(params).forEach(key => authUrl.searchParams.append(key, params[key]));
    window.location.href = authUrl.toString();
};

3. Fase de Recuperación de Datos (Respuesta)

Tras el login, Google devuelve un Access Token en la URL. Usamos este token para consultar la API de Userinfo.

ENDPOINT: https://www.googleapis.com/oauth2/v3/userinfo

Código: entrada.html (JavaScript)

const token = new URLSearchParams(window.location.hash.substring(1)).get('access_token');

if (token) {
    fetch(`https://www.googleapis.com/oauth2/v3/userinfo?access_token=${token}`)
        .then(res => res.json())
        .then(data => {
            console.log("Perfil obtenido:", data);
            renderizarUsuario(data);
        });
}

4. Resultado de la API (Formato JSON)

La API de Google responde con un objeto JSON que contiene la identidad procesada del usuario. Este formato permite una integración limpia con el DOM:

{
  "sub": "1234567890",
  "name": "Jonathan Perez",
  "picture": "https://lh3.googleusercontent.com/a/...",
  "email": "jonaperez0406@gmail.com",
  "email_verified": true
}

5. Resumen de Tecnologías Utilizadas

  • Google OAuth 2.0: Protocolo de autorización.
  • Google Userinfo API: Servicio de datos de perfil.
  • Fetch API: Método asíncrono para consumir la respuesta JSON.
  • Live Server: Entorno de ejecución local (127.0.0.1:5500).
